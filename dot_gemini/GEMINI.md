# Guiding Principles
- **Philosophy**: DRY (Don't Repeat Yourself), KISS (Keep It Simple, Stupid).
- **Type Safety**: Use strict typing. No `any` types without explicit justification.
- **Error Handling**: Anticipate edge cases.
- **Modern Standards**: Use current best practices and up-to-date tooling.

# Tech Stack
- **Python**: Use `uv`.
- **Node**: Check lockfile first (`pnpm-lock.yaml` → pnpm, `bun.lockb` → bun).

# Code Style
- Concise, readable code over verbose documentation
- Self-documenting variable/function names
- Comments explain *why*, not *what*
